# New Branch and PR Skill Context

## Project Information
- **Name**: Lien Automation v2
- **Repository**: https://github.com/Jmx097/lien-automation-v2
- **Default Branch**: main
- **Git Workflow**: Feature branches with pull requests

## Branch Naming Convention
Use descriptive names that indicate the purpose of the branch:
- `feature/add-new-scraping-capability`
- `bugfix/fix-pdf-parsing-issue`
- `refactor/improve-performance`
- `update/dependency-upgrades`

## Commit Message Guidelines
Follow conventional commit format:
- `feat: Add new scraping capability for XYZ`
- `fix: Resolve PDF parsing issue`
- `refactor: Improve performance of data processing`
- `docs: Update README with new instructions`

## Pull Request Process
1. Ensure all tests pass locally before creating PR
2. Include a clear description of changes
3. List any breaking changes
4. Add test instructions if applicable
5. Link to any related issues

## Code Review Expectations
- Changes should be focused and well-scoped
- Include appropriate tests for new functionality
- Follow existing code style and patterns
- Maintain backward compatibility when possible

## Pre-PR Checklist
- [ ] All tests pass locally
- [ ] Code follows project style guidelines
- [ ] Documentation updated if needed
- [ ] No sensitive information in commits
- [ ] Branch is up to date with main
